# ✅ ИСПРАВЛЕНИЕ: Автоматический редирект на отчёт

## Проблема
После завершения анализа пользователь НЕ перенаправлялся на страницу отчёта.
Показывалась ошибка: `GET /api/analysis/report/undefined 404`

## Причина
API endpoint `/api/analyze` возвращал поле `analysisId`, 
но клиентский код искал поле `id`.

## Решение

**Файл:** `app/api/analyze/route.ts` (строка ~240)

**Было:**
```javascript
return NextResponse.json({
  analysisId: analysis.id,
  ...
});
```

**Стало:**
```javascript
return NextResponse.json({
  id: analysis.id,           // Добавлено для router.push
  analysisId: analysis.id,   // Оставлено для совместимости
  ...
});
```

**Также исправлено:** `app/analysis/page.tsx` (строка 110)
- Синтаксис `router.push()` исправлен на правильные круглые скобки

## Результат
✅ После завершения анализа автоматически открывается страница отчёта
✅ `router.push(\`/report/${data.id}\`)` теперь работает корректно
✅ Функционал "Скрипт первого касания" доступен в отчёте
